# 다음 대화 시 확인 사항

**사용자 요청 (세션 종료 전):**  
"이후 다시 이 창을 띄워서 너에게 물어볼 때, **내가 도커 설정 다 했는지를 확인해 주고** 일을 시작해 줘."

---

## 작업 시작 전 확인할 것

1. **Synology NAS Docker(ELS 백엔드) 설정 완료 여부**
   - NAS에서 `els-backend` 이미지 빌드 및 컨테이너 실행을 했는지
   - 포트 2929 노출 및 동작 확인 (`/api/els/capabilities` 호출 시 `available: true` 반환하는지)
   - (선택) DDNS + 역방향 프록시로 외부 URL 노출했는지

2. **웹 쪽 연동**
   - Next.js(또는 Vercel)에 `ELS_BACKEND_URL` 환경 변수를 설정했는지
   - 설정했다면 해당 URL로 프록시가 정상 동작하는지

3. **확인 방법**
   - 사용자에게 "NAS Docker(ELS 백엔드) 설정을 이미 완료하셨나요? 완료하셨다면 접속 URL(예: http://NAS_IP:2929 또는 https://DDNS/els-api)을 알려주시면 연동 상태를 점검하겠습니다." 등으로 물어본 뒤, 필요 시 `NAS_DOCKER_ELS.md` 기준으로 점검하고 이어서 작업 진행.

---

**참고:** Docker 설정 가이드는 `NAS_DOCKER_ELS.md` 에 정리되어 있음.
