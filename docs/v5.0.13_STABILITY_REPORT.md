# 📊 ELS Solution v5.0.13 안정화 리포트

## 🚀 최근 업데이트 요약 (v5.0.10 ~ v5.0.13)

### 1. 지능형 검색 엔진 고도화
- **항구 키워드 가중치(+20점)**: "부산신항", "인천항", "평택항" 매칭 시 우선순위 부여.
- **검색 유연화**: 단어 개수가 많아도 핵심 구간을 찾아내도록 임계치 하향 (40 -> 35).
- **편도/할증 로직**: 의왕 ICD 기반 편도 검색 및 인천 터미널 반납 할증(4만원) 자동 계산.

### 2. 인프라 및 통신 안정화
- **환경변수 세척 (v5.0.11)**: `web/.env.local`의 윈도우 줄바꿈(`\r`)이 API Key를 오염시키는 문제 해결.
- **프록시 자동 대응 (v5.0.13)**: Vercel에서 NAS로 접근할 때 HTTP/HTTPS 주소를 상황에 맞게 폴백.
- **시간대 동기화 (v5.0.12)**: 도커 로그 시간을 한국 표준시(KST)로 일치.

### ⚠️ 빌드 지연 안내 (Dockerfile 수정)
- **사유**: 로그 시간 정상화를 위해 `tzdata` 패키지 설치 및 시간대 설정을 Dockerfile에 추가함.
- **영향**: 도커 이미지의 레이어 캐시가 갱신되어, 한 번은 전체 빌드(30~40분)가 발생함.
- **대책**: 이후부터는 소스 코드만 변경되므로 다시 1분 이내의 빠른 빌드가 가능함.
