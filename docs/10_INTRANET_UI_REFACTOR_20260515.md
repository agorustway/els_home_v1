# 인트라넷 연락처형 페이지 UI 리팩토링 기록 (2026-05-15)

## 목표
- 형이 첨부한 eTrans 화면처럼 업무자가 빠르게 훑는 **고밀도 데이터 화면**을 기준으로 인트라넷 연락처 계열을 정렬한다.
- 데이터 구조와 API는 유지하고, 목록/상세/버튼/모바일 접근 흐름만 안정적으로 통일한다.
- 이후 신규 연락처형 페이지를 만들 때 같은 파이프라인을 재사용할 수 있게 공통 컴포넌트와 CSS 기준점을 둔다.

## 적용 범위
- 사내연락망: `/employees/internal-contacts`
- 외부연락처: `/employees/external-contacts`
- 협력사정보: `/employees/partner-contacts`
- 운전원정보: `/employees/driver-contacts`
- 작업지정보: `/employees/work-sites`
- 랜덤게임/송미관은 공식 메뉴가 아닌 이스터에그로 유지

## 공통 파이프라인
1. **Header**
   - 흰색 업무 패널, 좌측 3px 블루 포인트, 우측 버튼군.
   - 등록/수정/삭제/목록 버튼 높이와 색상 통일.
2. **Filter**
   - `ContactFilterBar`에서 검색/구분 필터를 공통 CSS 클래스로 정리.
   - 모바일에서는 필터를 1열로 접어 가로 공간을 확보.
3. **Desktop Table**
   - `IntranetDataTable` 공통 컴포넌트 사용.
   - `colgroup` 기반 고정 폭 컬럼과 마지막 유동 컬럼 조합.
   - 테이블 헤더는 짙은 블루, 행은 얇은 구분선과 hover만 사용.
4. **Mobile Card**
   - 768px 이하에서 테이블 숨김, 카드 목록 노출.
   - Galaxy S24 360px 기준 좌우 여백 3~4px, 카드 padding 8~9px.
   - 이름/상태/전화/보조정보/메모 순서로 일관화.
5. **Detail**
   - `DetailHero → DetailGrid → DetailSection` 구조로 통일.
   - 상세 필드 라벨/본문 크기 고정.
   - 첨부파일은 `AttachmentList`로 같은 행 구조 사용.

## 추가한 공통 파일
- `web/components/IntranetDataTable.js`
  - 공통 데이터 테이블, 전화 링크, 배지, 이니셜/이미지 아바타.
- `web/components/IntranetRecordDetail.js`
  - 상세 히어로, 상세 그리드, 상세 필드, 섹션, 첨부 리스트.
- `web/utils/contactDisplay.js`
  - 전화번호 포맷, 날짜 포맷, 안전 파일 URL, 이미지 파일 판별, 값 조합.

- `npm.cmd run build`: 통과
- `contactDisplay` 빠른 케이스 검증: 통과

## 빌드 참고
- 빌드 중 일부 외부 HTTPS 호출은 샌드박스 네트워크 제한으로 `EACCES` 경고가 출력됐다.
- Next 빌드는 정상 종료했고, 연락처형 페이지의 컴파일/라우팅 오류는 발생하지 않았다.

## 남은 후보 작업
- 신규/수정 폼 내부의 일부 드래그 업로드/사진 선택 영역에는 아직 레거시 인라인 스타일이 남아 있다.
- 현재 구조는 CSS 오버라이드로 충분히 정렬되지만, 다음 리팩토링에서는 폼 전용 `IntranetFormLayout`까지 분리하면 더 단단해진다.
- 인증이 필요한 화면이라 실제 운영 데이터 기반 모바일 스크린샷 QA는 로그인 세션이 있는 브라우저에서 한 번 더 확인하는 것이 좋다.

## v5.13.1 추가 보정
- 랜덤게임 라우트를 복구하고 공식 메뉴 미노출 이스터에그로 문서화했다.
- 송미관 모달은 뉴스 페이지 하단 숨은 트리거로 유지하며, 공식 페이지 목록에는 넣지 않는다.
- 작업지정보 목록의 주소 폭을 줄이고 주의사항/특이사항 축약 컬럼을 추가했다.
- 업무자료실 상세의 첨부파일은 목록형 테이블로 변경했다.
- `SiteLayout.module.css`에 인트라넷 전체 폰트/버튼 높이 보정 레이어를 추가했다.

## v5.13.3 추가 보정
- 업무보고 목록/상세/작성 계열을 연락처형 페이지와 같은 짙은 블루 테이블 헤더, 0.74~0.8rem 표 셀 기준으로 맞췄다.

- 아산 선적관리 행 높이를 28px로 낮추고 셀 padding을 줄여 선적관리 테이블이 현재 폰트 크기에 맞는 밀도로 보이게 조정했다.
