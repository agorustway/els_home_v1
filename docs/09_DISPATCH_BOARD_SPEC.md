## 1. 배차판 원본 파일 구조

- **파일**: `2026년_배차-일일배차(글로비스KD외).xlsm`, `2026년_배차-일일배차(모비스AS).xlsm`
- **시트**: 날짜별 (예: `5.9`, `5.11`)
- **DB 테이블**: `branch_dispatch` (branch_id='asan', type='glovis'|'mobis')
- **동기화**: NAS 파일 변경 감지 → Fast Sync (±15일), 매일 04:00 Full Sync
- **누적 보존**: 원본 파일에서 날짜 시트를 삭제해도 DB 레코드는 삭제하지 않는다. 삭제된 시트는 변경이 끝난 마감 스냅샷으로 보고 웹 조회 원장에 보존한다.

## 2. 핵심 컬럼 의미 (가장 중요)

| 컬럼명 | 의미 | 예시값 |
|--------|------|--------|
| **오더(계)** | 화주가 요청한 전체 오더 합산 수량 (배차 완료 여부와 무관한 "주문량") | `5` |
| **배차정보** | 화주 요청 도착 시간 (화주가 "몇 시에 도착해달라"는 요청) | `08 09 10` |
| **기타/아산/부산/인천/광양/평택/중부/부곡** | **상차지(출발지) 컬럼** — 이 컬럼의 값이 실제 배차 내용 | `이지4,대신1` |
| **셀 메모(comments)** | 관리자가 배차 확정 후 기입한 **실제 배차 시간 + 업체** | `이지 13 14 15 16, 대신: 10` |
| **담당자** | 담당 직원명 (예: 양진영, 강주희) | `양진영` |
| **작업지** | 화주 작업 장소 (예: 글로비스1포장장, 아산수출물류센터) | `글로비스2포장장(오도렐)` |
| **고객사(국가)** | 화주명 또는 수출 국가 | `미국`, `KaGA` |

## 3. 지역 컬럼 값 파싱 규칙 ⭐ (핵심)

```
지역 컬럼(예: 부산) 값: "이지4,대신1"
→ 이지 업체 4대 + 대신 업체 1대 = 합계 5대 (부산 상차)

숫자가 없으면 1대로 해석:
"자차" → 자차 1대
"이지3" → 이지 3대
"대신1,자차" → 대신 1대 + 자차 1대 = 2대
```

---

## 4. 메모 시간 해석 규칙 ⭐ (핵심)

```
셀 메모 예: "이지 13 14 15 16, 대신: 10"
→ 이지 업체: 13시 / 14시 / 15시 / 16시 배차 (4대)
→ 대신 업체: 10시 배차 (1대)

"10시 배차 몇 대야?" 질문 시:
→ 메모에 '10'이 포함된 행 검색
→ 해당 행의 지역 컬럼에서 대신 업체 찾기
→ 답변: "대신 1대 (부산 상차, 10시 배차)"
```

## 5. 오더 vs 배차 구분

| 구분 | 컬럼 | 의미 |
|------|------|------|
| **오더 수량** | `오더(계)` | 화주 요청 전체량 — 배차 완료와 무관 |
| **배차 수량** | 지역 컬럼(부산/아산 등)의 업체별 합산 | 실제 배차된 양 (오더 ≤ 배차 가능) |

> ⚠️ 배차가 100% 완료되지 않을 수 있다. 오더 = 5대이지만 배차 = 3대일 수 있음.

## 6. 질문 유형별 AI 답변 기준

| 질문 유형 | 검색 기준 | 답변 형식 |
|----------|----------|----------|
| "오늘 배차 몇 대?" | 오늘 날짜 레코드 전체 지역 컬럼 합산 | "글로비스 N대 / 모비스 N대 = 총 N대" |
| "10시 배차 몇 대?" | 메모에 '10' 포함된 행, 해당 지역 컬럼 집계 | "업체A N대, 업체B N대 (총 N대)" |
| "부산 배차 몇 대?" | 부산 컬럼에 값 있는 모든 행 합산 | "이지 4대, 대신 1대 (부산 총 5대)" |
| "이지 배차 몇 대?" | 지역 컬럼에 '이지N' 패턴 전체 합산 | "이지 총 N대" |
| "양진영 배차?" | 담당자 컬럼 = '양진영'인 행 집계 | "담당자 양진영: N건 / N대" |
| "미국 배차 몇 대?" | 고객사/국가 컬럼에 '미국' 포함된 행 집계 | "미국향 N대" |
| "5월9일 배차?" | target_date = '2026-05-09' 레코드 | "5/9일: 글로비스 N대 / 모비스 N대" |
| "월요일 배차?" | 해당 월요일 날짜 레코드 | 날짜별 소계 표시 |

## 7. AI RAG 주입 구조 (route.js 기준)

- **현재 구현**: `web/utils/asanDispatchRag.mjs`
- **쿼리 범위**: 특정 날짜/내일/어제/요일/주간/월간 의도를 먼저 해석하고 Supabase `branch_dispatch`를 조회
- **도표형 스키마 선주입**: DB 헤더와 셀 값 패턴으로 `오더 컬럼`, `픽업지역/상차지 컬럼`, `메모/시간 컬럼`, `기본정보 컬럼`을 먼저 요약
- **주입 형식**: 전체 날짜별 현황 → 질문 조건 매칭 현황 → 매칭 상세 행 순서로 주입
- **필터링 주의**: 날짜/요일/차번/집계어/작업지 질문어는 검색 키워드에서 제외하고, 실제 지역명·업체명·작업지명만 조건으로 사용
- **확장 기준**: 새 픽업지역 컬럼은 지역명 하드코딩만 보지 않고 `업체명+대수` 셀 패턴으로 추론
