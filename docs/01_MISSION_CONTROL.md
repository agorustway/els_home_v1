## 📋 최근 변경 (v4.2.42 — 2026-03-30)
- **실시간 관제 명령 체계 구축**: 웹 관리자 페이지에서 '실시간' 버튼 클릭 시, `/api/vehicle-tracking/emergency` API를 통해 `SYSTEM_COMMAND`를 전송하고, 앱이 이를 30초 폴링 중 가로채어 즉각 고정밀 수집 모드로 전환하도록 구현.
- **Adaptive GPS Interval**: 평시에는 속도별(30~60초) 수집을 유지하다가, 관리자 명령 수신 시에만 **3초 고정 주기**로 강제 전환하여 배터리 수명과 관제 정밀도의 균형을 맞춤.
- **UI 시각화 개선**: 실시간 모드 진입 시 앱 상단 및 오버레이에 `실시간 수집중` 문구를 노란색(Orange-500)으로 표시하여 드라이버가 관제 상태를 인지할 수 있도록 보완.
- **세션 타임아웃 조정**: 실시간 관제 부하를 줄이기 위해 기존 2분에서 **1분(60초)**으로 자동 종료 시간을 단축.

## 📋 최근 변경 (v4.2.41 — 2026-03-29)
- **음영지역 대응**: 터널 진입 등으로 GPS 수신이 30초 이상 끊길 경우, 5초 주기로 즉시 재수신을 시도하는 Wake-up 로직 강화.
- **가짜 위치 차단**: 기지국(LBS)이나 Wi-Fi 기반의 오차 큰 위치 데이터가 튀는 현상을 막기 위해 속도(speed)값이 null인 데이터 필터링.
