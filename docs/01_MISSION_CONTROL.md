- **[FEAT] NAS 지능형 확장자 통계 도입**: NAS 파싱 현황 브리핑 시 지점별 진행도 외에 PDF, XLSX, DOCX 등 주요 확장자별 개수 집계 기능 추가 (v5.8.0).
- **[FEAT] 실시간 날씨/미세먼지 통합 트리거**: 사용자의 기상 질문 시 날씨 정보와 미세먼지(K-Skill)를 한 번에 조회하여 제공하도록 로직 최적화 (v5.8.0).
- **[FEAT] 아산 배차판 듀얼 스피드 동기화 도입**: NAS 부하를 줄이기 위해 "오늘(±15일)은 즉시, 전체는 10분마다" 동기화하는 스마트 스케줄링 적용 (v5.5.13).
- **[STABILITY] 데이터 증발 방지(UPSERT) 적용**: 기존 Delete 후 Insert 방식에서 UPSERT 방식으로 전환하여 동기화 중 에러 발생 시에도 기존 데이터가 유지되도록 보강.
- **[SOLVED] 아산 배차판 전체 동기화 복구**: 형의 요청에 따라 날짜 제한 없이 모든 시트 동기화로 원복 (v5.5.12).
- **[STABILITY] 아산 배차판 메모리 최적화**: `read_only=True` 및 `gc.collect()` 도입으로 수십 개 시트 처리 시 OOM 해결 (v5.5.8).
- **[STABILITY] NAS 벡터화 파이프라인 안정화 패치**: Supabase 429 Rate Limit 대응 재시도 로직, 좀비 락 방지(2시간 초과 시 해제) 및 쉘 스크립트 타임아웃 보강 완료.
- **[FEAT] 모바일 대화 목록(History Drawer) 추가**: 모바일 환경에서 헤더 아이콘 클릭 시 과거 대화 목록 접근 가능하도록 드로어 UI 구현.
- **[UI/UX] AI 가이드 UI 정제 및 공식 호칭 적용**: 가이드 패널 내 이모지 제거, 카드 디자인 통일 및 호칭 변경("형" -> "사용자"). v5.3.3 반영.
- **[FIX] NAS 백엔드 실행 오류 핫픽스 (v5.9.6)**: v5.9.3 업데이트 시 누락된 `web_vectorizer.py` 파일을 Docker 설정에 긴급 추가하여 `els-core` 기동 불가 문제 및 배차 동기화 중단 현상 해결.
- **[FEAT] NAS 정밀 검색(Precision Query) 도입 (v5.9.5)**: 차량번호(3~4자리) 및 특정 월(3월, 4월 등) 검색 시 벡터 검색 대신 DB 직접 쿼리를 수행하여 100% 정확한 데이터 추출 보장.
- **[FEAT] 파일명 별칭(Alias) 매핑**: "수출리스트", "마감자료", "배차판" 등 고유 명칭과 실제 파일명 간 자동 매칭 로직 추가.
- **[FEAT] 웹 게시판 첨부파일 벡터화 (Web Attachment Vectorizer) 도입**: `posts` 및 `work_docs`에 업로드된 파일(S3/Supabase Storage)을 자동으로 다운로드하여 벡터화하는 로직 추가 (v5.9.3).
- **[FEAT] NAS 최신 자료 우선 가중치(Recency Boost) 로직 도입**: 파일명/시트명 내 연도 및 월 정보를 분석하여 최신 자료에 가중치를 부여하고, 모호할 경우 여러 후보를 동시에 제안하도록 AI 지침(Rule 17) 추가 (v5.9.2).
- **[FEAT] NAS 문서 검색 재정렬(Re-ranking) 로직 도입**: 시맨틱 검색 결과 중 파일명이 사용자의 검색어와 일치하는 항목에 가중치를 부여하여 검색 정확도 대폭 개선 (v5.9.1).
- **[FEAT] NAS 문서 검색 신뢰도 대폭 강화 (Hybrid Search)**: 시맨틱 검색(벡터) 실패 시 파일명 키워드 기반 직접 매칭 로직 추가, 검색 임계값(Threshold) 하향(0.55->0.45) 및 트리거 키워드 대폭 확장(현황, 배차, 실적 등). (v5.9.0)
- **[FIX] AI 채팅 시스템 지시문 문법 오류 수정**: `BASE_SYSTEM_INSTRUCTION` 내 중첩 백틱으로 인한 Vercel 빌드 실패 해결 (v5.8.9).
- **[FEAT] AI 채팅 진입 시 새 대화 자동 시작 로직 적용**: 페이지 진입/새로고침 시 기존 대화는 히스토리에 보존하고, 항상 깨끗한 '새로운 대화'에서 시작하도록 개선 (v5.8.8).
- **[FEAT] 안전운임 ICD 연계 다중 제안 로직 도입**: "의왕-부산신항" 등 특례 구간 문의 시 '의왕 편도 / 인천 반납 / 일반 왕복' 3종 세트를 능동적으로 제안하도록 AI 지시사항 및 검색 가중치 고도화 (v5.8.7).
- **[SOLVED] 안전운임 최단경로 법규 로직 적용**: 제3조(거리계산 및 운임적용) 규정에 따라 '최적경로'와 '큰길우선' 중 짧은 거리 자동 선택 구현.
- **[UI/UX] 안전운임 엑셀 다운로드 개선**: 엑셀 출력 시 각 운임 섹션별 기점/행선지 명시적 표기 및 스타일 개선으로 가독성 향상.
