# 데스크탑·안드로이드 앱 (컨테이너 이력조회)

nollae.com(Vercel)에서는 Python·Chrome이 없어 **컨테이너 이력조회**를 사용할 수 없습니다.  
데스크탑·안드로이드에서 사용하려면 **설치형 앱**이 필요합니다.

---

## 1. 데스크탑 (Windows 인스톨러)

### 방식
- **Electron** 앱이 로컬에서 다음을 수행합니다.
  1. **로컬 HTTP 서버** 실행 (예: 포트 2929)
  2. **컨테이너 이력조회 UI** 제공 (웹과 동일한 화면)
  3. **API** (`/api/els/login`, `run`, `parse-xlsx` 등)를 **Node에서 Python/elsbot 호출**로 구현
- 사용자는 **Python**과 **Chrome**을 PC에 설치해 두어야 합니다. (추후 Python 내장 번들 가능)

### 설치 후 사용 흐름
1. **ELS 컨테이너 이력조회** 앱 실행
2. 앱 창에서 로그인 → 컨테이너 입력/업로드 → 조회·다운로드 (웹과 동일)
3. 같은 PC에서만 사용 시 여기서 끝

### 인스톨러 생성
- `desktop/` 폴더에서 `npm run build` 후 `npm run dist` (electron-builder)
- Windows용 `.exe` 인스톨러가 `desktop/dist/` 에 생성됩니다.

---

## 2. 안드로이드

### 제약
- ELS 조회는 **Selenium + Chrome**으로 동작합니다. 이 조합은 **PC/서버**에서만 실행 가능합니다.
- 안드로이드 단말에서는 **Chrome 자동화(셀레니움)** 를 같은 방식으로 돌릴 수 없습니다.

### 실사용 가능한 방식: **데스크탑 앱을 서버로 쓰기**
1. **데스크탑 앱**을 “서버 모드”로 실행 → 같은 Wi‑Fi의 다른 기기에서 접속 가능하도록 포트 개방 (예: `0.0.0.0:2929`)
2. **안드로이드 앱**은 **WebView**로, “접속 주소”만 입력 (예: `192.168.0.10:2929`)
3. 입력한 주소로 접속하면 **컨테이너 이력조회 페이지**가 그대로 열리고, API도 데스크탑 앱이 처리합니다.
4. 즉, **안드로이드 = 클라이언트**, **데스크탑 = 서버** 역할입니다.

### 안드로이드용 “바이너리(APK)” 의미
- **APK**: WebView + 주소 입력 화면 (데스크탑 PC IP:포트 입력)
- 같은 Wi‑Fi에서 데스크탑 앱을 먼저 실행해 두면, 안드로이드에서 해당 주소로 접속해 **동일한 컨테이너 이력조회** 사용 가능  
- 별도 백엔드 서버 없이, **데스크탑 한 대**만 있으면 됩니다.

---

## 3. 요약

| 환경 | 사용 방법 |
|------|-----------|
| **nollae.com (배포)** | 컨테이너 이력조회 **불가** (안내 배너만 표시) |
| **데스크탑 (Windows)** | **데스크탑 앱 인스톨러** 설치 → Python·Chrome 설치 → 앱 실행 후 조회 |
| **안드로이드** | **데스크탑 앱**을 같은 Wi‑Fi에서 “서버 모드”로 실행 → **안드로이드 앱(WebView)** 에서 PC 주소 입력 후 접속하여 조회 |

즉, **데스크탑·안드로이드 모두 “해당 페이지”를 사용할 수 있게 하려면**  
- **데스크탑**: Windows용 인스톨러(바이너리) 제공  
- **안드로이드**: 데스크탑에 연결하는 WebView APK 제공  
으로 해결할 수 있습니다.
